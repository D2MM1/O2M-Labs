How It Works

A step-by-step walkthrough of the interface, validation logic, and edge cases the tool handles automatically.

Guide Interface walkthrough

1

Upload CSV file

Click the Choose File button and select a .csv file from your computer or smartphone. The file is read entirely in your browser — nothing is sent to a server.

2

Paste CSV here

Alternatively, copy CSV data from a spreadsheet or editor and paste it directly into the text area. This works for quick tests or snippets copied from other tools.

3

Input delimiter

The input delimiter is the character that separates columns in your file. Choose from comma ,, semicolon ;, or tab. Select Auto detect to let the tool guess based on the first row, or choose Custom to enter any character (e.g. |).

4

Output delimiter

When the Fix CSV button is used, the corrected file is written using this delimiter. This lets you convert a malformed CSV into a clean one with your preferred separator.

5

Optional rules

Extra validation checks applied on top of the basic structure check.

All columns required — every row must have the same number of columns as the header row.

Regex column 1 — apply a regular expression to the first column (e.g. ^[A-Za-z]+$ ensures only letters).

Min length column 2 — require the second column to be at least a certain number of characters long (e.g. 3 means "ab" fails, "abc" passes).

If any rule fails, the row is marked with a descriptive error in the fixed CSV.
6

Validate CSV

Click the Validate CSV button to run all checks. Results appear immediately below the form — a green summary for valid files, or a detailed error report for invalid ones.

Core How validation works

Step by step:
  1. Your CSV is parsed using the selected input delimiter. Quoted fields like "Hello, world" are handled correctly — commas inside quotes are not treated as column separators.
  2. The tool checks column consistency: the first row (header) sets the expected column count. Any row with a different number of columns is flagged.
  3. If optional rules are enabled, each data row is checked against them. Failures are recorded per row.
  4. If valid — a green message appears and the original CSV is shown unchanged. No modifications, no extra columns.
  5. If invalid — a list of problem rows is displayed. A Fix CSV button appears. Clicking it generates a corrected file with an extra Error column describing what went wrong on each affected row.

Detail Edge cases handled

  • Delimiter mismatch: if your data uses semicolons but comma is selected, validation will fail and the tool will suggest the correct delimiter.
  • Quoted fields with embedded delimiters: e.g. "Smith, John", 30 is parsed as two columns, not three.
  • Empty rows are automatically skipped during validation.
  • Large files: all processing is local, so performance depends on your device. Files up to 10 MB work smoothly on most machines.
Privacy note — Your CSV stays entirely within your browser. No data is sent to any server and all processing happens locally on your device. For more details, visit the FAQ page.
